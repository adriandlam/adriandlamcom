I built this to understand how neural networks actually work. The math behind a multi-layer perceptron is surprisingly simple — matrix multiplies, ReLU, softmax, chain rule — but frameworks like PyTorch abstract all of it away behind `nn.Module` and autograd. So I wrote everything from scratch: forward pass, backpropagation, gradient descent, all of it. No `torch.nn`, no automatic differentiation.

I also trained a CNN using PyTorch for comparison, to see how much spatial awareness matters for image classification.

## Architecture

### MLP (from scratch)

A 4-layer fully connected network:

- **Input**: 784 neurons (28×28 pixels, flattened)
- **Hidden layers**: 3 × 256 neurons with ReLU
- **Output**: 10 neurons with softmax

Each hidden layer uses a dual-bias design — a pre-activation bias before ReLU and a post-activation bias that bypasses it. This gives each neuron a small linear skip connection past its own activation function, which I found improved accuracy slightly.

### CNN

The CNN uses spatial structure that the MLP throws away by flattening:

- **Conv1**: 32 filters (3×3), BatchNorm, ReLU, MaxPool
- **Conv2**: 32 filters (3×3), BatchNorm, ReLU, MaxPool
- **FC**: 800 → 10

Convolutional layers detect local patterns like edges and curves that are translation-invariant — a "7" gets recognized whether it's drawn in the center or corner.

## Training

**MLP** — trained for 5 epochs with Xavier initialization, learning rate 0.5 with cosine annealing, batch size 128. Cross-entropy loss implemented from scratch. The entire training loop — forward pass, backpropagation, weight updates — is written without any automatic differentiation. Gradients derived by hand and computed explicitly for each layer.

**CNN** — standard PyTorch training with Adam optimizer, dropout (0.25), 45 epochs.

## Results

| Model | Demo Accuracy | Parameters |
|-------|-------------|------------|
| MLP (from scratch) | 97.76% | ~397K |
| CNN (PyTorch) | 96.58% | ~18K |

The CNN in the [repo](https://github.com/adriandlam/mnist-classifier) achieves 99.31% when fully trained — the demo runs a lighter checkpoint to keep browser inference fast. Both models run entirely in your browser using pure TypeScript. No server, no ONNX, no WebAssembly. Just matrix math.
